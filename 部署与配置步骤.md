# 第二步、第三步 一步一步操作指南

下面按顺序做即可。**第二步**是把后端放到网上、**第三步**是让小程序能访问你的后端。

---

## 你现在应该干什么（按这个顺序做）

你已经把前后端都推到 GitHub（**https://github.com/Ccwevelyn/eat-what**）了，接下来：

1. **先做第二步**：用 Render 把你的**后端**从 GitHub 部署到网上，拿到一个后端地址（例如 `https://xxx.onrender.com`）。  
   → 从下面 **「方式 A：用 Render 部署（你的仓库已就绪）」** 开始，**从 A1 开始做**；其中 **A2 可以跳过**（不用再建仓库或上传代码），**直接从 A3 开始**，连你现有的 **eat-what** 仓库，并把 **Root Directory** 填成 **server**。

2. **再做第三步**：在小程序里填上这个后端地址，并在微信公众平台把域名加为 request 合法域名。  
   → 做完第二步后，按下面 **「第三步：配置小程序」** 一步步做。

3. **最后**：在微信开发者工具里点「随机选一家」试一下，能出餐厅就说明通了。

---

# 第二步：部署后端（让网上有一个你的接口地址）

后端需要 24 小时能在互联网被访问到，小程序才能随时用。有两种做法，**任选一种**即可。

---

## 方式 A：用 Render 免费托管（推荐，你的仓库已就绪）

你的代码已经在 **https://github.com/Ccwevelyn/eat-what**，前后端都在。Render 只要从这个仓库里**只部署 server 文件夹**即可。

### A1. 安装 Node.js（如果还没装）

1. 打开浏览器，打开：**https://nodejs.org/**
2. 点击左边绿色的 **LTS** 版本（推荐），下载安装包。
3. 双击下载好的安装包，一路点「下一步」安装（可以全部用默认选项）。
4. 安装完成后，按 `Win + R` 打开「运行」，输入 `powershell` 回车，打开 PowerShell。
5. 在窗口里输入下面这一行，回车：
   ```bash
   node -v
   ```
   如果出现类似 `v20.10.0` 的版本号，说明安装成功。

### A2. 跳过（你的代码已在 GitHub）

你已经把整个项目（含 **server** 文件夹）推送到 **Ccwevelyn/eat-what**，不需要再建新仓库或上传文件。**直接做 A3**。

### A3. 在 Render 上创建免费服务，连到你现有的 eat-what 仓库

1. 打开 **https://render.com/** ，点击 **Get Started for Free**。
2. 选择 **Sign up with GitHub**，按提示授权 Render 访问你的 GitHub（用 Ccwevelyn 账号登录并授权）。
3. 登录后，在 Dashboard 点 **New +** → **Web Service**。
4. 在 **Connect a repository** 里，找到 **Ccwevelyn/eat-what**（你的「今天吃啥」仓库），点右边的 **Connect**。
5. **Configure your service**（按下面逐项填）：
   - **Name**：可保持自动生成的，或改成 `eatwhat`。
   - **Region**：选 **Singapore**（离澳门近）。
   - **Runtime**：选 **Node**。
   - **Root Directory**：**一定要填** → 填 **`server`**（这样 Render 只会用仓库里的 server 文件夹来部署后端，不会去跑小程序代码）。
   - **Build Command**：留空。
   - **Start Command**：填 **`node index.js`**（必须一模一样）。
   - **Instance Type**：选 **Free**。
6. 点 **Create Web Service**。等 2～5 分钟，直到上面出现绿色 **Live**。
7. 页面上方会有一行 **URL**，例如：`https://eat-what-xxxx.onrender.com`。  
   **复制这个地址**（到 `.com` 为止，不要加 `/api`），后面第三步会用到。

---

## 方式 B：在自己电脑上运行 + 用内网穿透（不推荐长期用，但可先试）

适合先验证小程序能否连上后端，再考虑以后迁到云上。

### B1. 安装 Node.js

同 **A1**：打开 https://nodejs.org/ 下载 LTS，安装，在 PowerShell 里输入 `node -v` 能看到版本号即可。

### B2. 在电脑上启动后端

1. 按 `Win + R`，输入 `powershell`，回车。
2. 输入下面两行（每行输完按回车）：
   ```powershell
   cd C:\Users\Asus\Desktop\EatWhat\server
   node index.js
   ```
3. 看到类似 **「今天吃啥 后端运行在 http://localhost:3000（免费 OSM 数据，无需 API Key）」** 就说明后端已在本机运行。
4. **不要关掉这个窗口**，关掉后端就停了。

### B3. 用内网穿透得到一个 https 地址（微信要求必须是 https）

1. 打开 **https://ngrok.com/**，用邮箱注册免费账号。
2. 下载 ngrok（选 Windows），解压得到一个 `ngrok.exe`。
3. 在 ngrok 官网的 **Your Authtoken** 里复制你的 token。
4. 再开一个 PowerShell 窗口，进入你放 `ngrok.exe` 的文件夹，执行（把 `你的token` 换成刚才复制的）：
   ```powershell
   .\ngrok.exe config add-authtoken 你的token
   .\ngrok.exe http 3000
   ```
5. 窗口里会出现一行 **Forwarding**，后面有一个 **https://xxxx.ngrok-free.app**，复制这个地址（到 `.app` 为止，不要后面的路径）。

**注意**：免费版 ngrok 每次重启地址会变，所以适合临时测试。真要长期给大家用，建议用方式 A（Render）。

---

## 第二步小结：你现在应该有一个「后端地址」

- 方式 A：类似 **https://eatwhat-xxxx.onrender.com**  
- 方式 B：类似 **https://xxxx.ngrok-free.app**  

记下这个地址，**不要加 `/api`**，下一步会用到。

# 第三步：配置小程序（让小程序去连你的后端）

## 3.1 修改小程序里的 API 地址

1. 用 **记事本** 或 **Cursor / VS Code** 打开文件：  
   **C:\Users\Asus\Desktop\EatWhat\config.js**

2. 找到这一行：
   ```javascript
   API_BASE: 'https://你的后端域名.com/api',
   ```

3. 把 **`https://你的后端域名.com`** 换成你在第二步记下的地址（仍然不要加 `/api`，代码里已经写了 `/api`）。  
   例如你记下的地址是 `https://eatwhat-xxxx.onrender.com`，就改成：
   ```javascript
   API_BASE: 'https://eatwhat-xxxx.onrender.com/api',
   ```
   注意：**保留引号和逗号**，只改中间那段。

4. 保存文件（Ctrl + S）。

## 3.2 在微信公众平台添加「request 合法域名」

小程序只能请求你登记过的域名，所以必须在这里加一次。

1. 打开浏览器，打开：**https://mp.weixin.qq.com/** ，用你的小程序管理员微信扫码登录。

2. 左侧菜单点 **「开发」** → **「开发管理」**。

3. 在 **开发管理** 页面里，找到 **「开发设置」** 这一块，点进去。

4. 往下滚动，找到 **「服务器域名」**。

5. 点 **「修改」**（需要管理员扫码确认）。

6. 在 **request 合法域名** 这一行的输入框里，填**只填域名，不要协议和路径**：
   - 若你的地址是 `https://eatwhat-xxxx.onrender.com`，就填：**eatwhat-xxxx.onrender.com**
   - 若你的地址是 `https://xxxx.ngrok-free.app`，就填：**xxxx.ngrok-free.app**  
   （不要写 `https://`，也不要写 `/api`）

7. 点 **「保存并提交」**，等提示保存成功。

## 3.3 在微信开发者工具里勾选「不校验合法域名」（仅本地调试时）

真机预览或正式版会按 3.2 的域名校验；本地调试时如果还没保存域名，可以先跳过校验：

1. 打开 **微信开发者工具**，打开你的「今天吃啥」项目。
2. 右上角点 **「详情」**。
3. 在 **「本地设置」** 里，勾选 **「不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书」**。
4. 关闭详情窗口，重新编译一下（点工具栏的 **编译**）。

---

## 第三步小结

- **config.js** 里的 `API_BASE` 已经改成你的后端地址 + `/api`。  
- 微信公众平台里，**request 合法域名** 已经填了你的域名（只有域名，无 `https://`）。  
- 本地调试可勾选「不校验合法域名」；真机预览和发布时会用你填的合法域名。

---

## 怎么确认已经通？

1. 确保后端正在运行（Render 上显示 Live；或本机 PowerShell 里 `node index.js` 没关）。
2. 在微信开发者工具里打开「今天吃啥」，第一页点「开启定位」，再选一个距离，点「随机选一家」。
3. 若出现一家餐厅名字和地址（不是固定的模拟数据），说明已经连上你的后端、用的是 OSM 数据。
