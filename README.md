# 今天吃啥 · 微信小程序

面向 **澳门** 发布的、**大家都可以用** 的微信小程序：通过定位 + 距离选择随机推荐附近餐厅，或通过菜系转盘决定今天吃啥。  
**餐厅与距离数据来自 OpenStreetMap（Overpass API）**，完全免费、无需 API Key、不收费。

---

### 1. 微信开发者工具（必装）

## 四、配置说明（config.js）

打开根目录下的 `config.js`：

- **API_BASE**：你的后端接口基础地址。  
  - **不填或仍为占位**时：小程序使用模拟数据，方便本地调试。  
  - **填成你部署的后端地址**（如 `https://你的域名.com/api`）后，小程序会自动用 **OpenStreetMap** 的免费餐厅与距离数据。
- **DEFAULT_LAT / DEFAULT_LNG**：默认经纬度（已设为澳门），无定位时后端兜底使用。

**详细地址：高德逆地理（必配）**  
- 详细地址由 **高德逆地理 API** 生成（内地+澳门可用）。  
- 必须在后端配置 **`AMAP_KEY`**，否则详细地址可能为空。

## 五、你现在要做的（让详细地址显示出来）

1. **申请高德 Web 服务 Key**  
   - 打开 [高德开放平台](https://lbs.amap.com/)，注册/登录。  
   - 进入「控制台」→「应用管理」→「创建新应用」（名称随意）→ 在应用下「添加」Key。  
   - Key 类型选 **「Web 服务」**，提交后复制生成的 Key。

2. **在后端配置 AMAP_KEY**  
   - **若后端在 Render**：Dashboard → 你的 Service → **Environment** → 添加变量 **Key**=`AMAP_KEY`，**Value**=你复制的 Key → Save，然后点 **Manual Deploy** 重新部署。  
   - **若后端在本机**：启动前设置环境变量，例如 PowerShell：`$env:AMAP_KEY="你的Key"`，再运行 `node index.js`。

3. **无需改小程序**  
   - 小程序端不用改，只要后端配好 `AMAP_KEY` 并重新部署，附近随机 / 菜系转盘返回的餐厅就会有高德返回的可读详细地址。

4. **可选：中/英切换**  
   - 右上角已有「中 / 英」切换，语言会本地保存。

## 九、附近餐厅接口返回说明（为什么模拟时只有澳门）

### 接口怎么调、返回什么

- **请求**：小程序用你当前的经纬度调后端，例如：  
  `GET https://你的后端/api/nearby?lat=22.1987&lng=113.5439&radius=2000`
- **参数**：`lat` 纬度、`lng` 经度、`radius` 半径（米）。
- **返回**：JSON，例如：
```json
{
  "list": [
    {
      "name": "某餐厅",
      "region": "澳门",
      "address": "澳门 新马路 123 号",
      "distance": 0.85,
      "cuisine": "粤菜"
    }
  ]
}
```

- **数据来源**：后端用这一组 `lat/lng` 去 **OpenStreetMap（Overpass）** 查「这个点附近」的餐厅，不做城市限制。  
- **`region`（地区）**：来自 OSM 里该餐厅的 `addr:city`、`addr:suburb`、`addr:district` 等。有就显示，没有就为空。  
- **`address`（详细地址）**：来自 OSM 的 `addr:full` 或 `addr:street` + `addr:housenumber` 等拼接。  

所以「地区」和「地址」都是 **OSM 上这条数据写的是什么就显示什么**，没有单独再调一个「地点 API」。

### 为什么模拟时只有澳门、地区只显示澳门？

因为在 **微信开发者工具** 里，**模拟位置** 默认或你选的是 **澳门**。  
流程是：

1. 小程序调用 `wx.getLocation` → 拿到的是 **模拟器给的经纬度**（澳门）。
2. 请求 `/api/nearby?lat=...&lng=...&radius=...` 时带的就是 **澳门** 的经纬度。
3. 后端按「澳门附近」去 Overpass 查 → 返回的都是 **澳门周边** 的餐厅。
4. 这些餐厅在 OSM 里很多都标了 `addr:city = 澳门`，所以 **地区** 就会显示「澳门」。

要想看到 **珠海或其他城市** 的餐厅和地区：

- **在模拟器**：在开发者工具里把 **模拟位置** 改成珠海（例如 纬度 22.27、经度 113.58），再点「随机选一家」。
- **在真机**：在珠海打开小程序、授权定位，就会用真实经纬度请求，返回的就是珠海附近的餐厅，地区会显示 OSM 里标的珠海相关名称（如香洲区等）。

## 十、常见问题

- **模拟器里点「随机选一家」没反应？**  
  请先在第一页点「开启定位」，在模拟器里可模拟定位（开发者工具右上角「…」→ 调试 → 模拟位置）。

- **没有 AppID 能运行吗？**  
  可以选「测试号」或「不使用 AppID」在本地模拟器运行，但不能真机预览和发布。

- **想加更多菜系？**  
  在 `pages/wheel/wheel.js` 里修改 `CUISINES` 数组，并保证转盘是 8 格的话，若增加/减少数量需要同步改 `pages/wheel/wheel.wxss` 里转盘扇形的数量与角度（目前按 8 格 45° 写的）。
